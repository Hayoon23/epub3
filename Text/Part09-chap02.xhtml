02 이동 탭 고급 활용

가 TCP 위치의 정량 이동

실무에서 웨이포인트의 정보 입력을 위해서 대략적인 위치 이동에 편리한 프리드라이브 활용만큼이나 정량적인 이동이 요구되는 경우가 많다. 이런 경우 <그림 IX-4>의 이동 탭의 정량 이동 기능을 알아 두어야 한다. <그림 IX-4> (a)의 ①을 이용하여 TCP의 위치를 정량적으로 이동할 수 있다. 이 영역의 숫자 입력 칸 가운데 아무 곳이나 클릭하면 <그림 IX-4> (b)의 화면으로 이동하게 된다. ①로 표시된 영역의 숫자를 필요에 따라 수치 입력하여 정확한 위치의 설정이 가능하다.

입력시 주의 사항이 있다. 첫 번째로 반드시 특징 정보를 먼저 확인하여야 한다는 것이다. <그림 IX-4> (b)의 경우 특징이 Base 조인트로 설정되어 있다. 혹시 이것이 뷰 등으로 설정되어 적절히 않다고 수정해야 한다. 두 번째로 단위이다. TCP의 위치 정보는 기본적으로 mm 단위를 사용한다. 흔히 하는 실수는 cm와 혼돈하여 입력하는 경우인데, 이렇게 되면 10배의 차이가 생기므로 동작 범위 밖으로 쉽게 벗어나게 된다.

정량 이동 위치를 입력할 때 현재 위치에서 특정 방향으로 정량 거리만큼 이동이 필요한 경우가 많다. 이 경우는 바로 위치 정보를 입력하는 것은 좋지 않다. <그림 IX-4>에서 보는 것과 같이 화면 크기의 제약으로 우리가 입력이 가능한 정밀도는 소수점 2자리 정도 수준이다. 하지만 실제 로봇 프로그램에서 웨이포인트 정보를 확인하면 소수점 10자리를 초과하는 정밀도로 정의됨을 확인할 수 있다. 예를 들어 현재의 로봇이 내부적으로 인식하고 있는 정확한 TCP X축의 위치가 –100.00123456789mm라면, +X축으로 100mm만큼 이동하기를 원해서 0mm를 입력하면 실제로 0.00123456789mm만큼 더 이동하는 결과가 발생한다. 이것은 생각보다 치명적일 수 있는데, 반복 동작하는 과정에서 그 오차가 누적될 수 있기 때문이다. 이때는 <그림 IX-4> (c) ①의 버튼을 이용하는 것이 유리하다. 이 버튼을 누르는 현재 위치에서 이동할 거리를 자동으로 연산하여 적용하므로, 직접 입력에 따른 오차 발생의 위험을 없앨 수 있기 때문이다.

앞에서 설명한 TCP 위치에서 단위 실수로 오입력하는 경우만큼이나 많이 하는 실수가 조인트의 방향을 의미하는 회전 값인데, 기본 단위는 라디안이다. 혹시 우리가 일상에서 몇 도로 표현하는 도(Degree) 단위를 많이 사용하기 때문에 초보자들의 실수가 잦은 편이다. 주의하도록 하자. <그림 IX-4> (d)는 방향을 설정하는 옵션으로 회전벡터 혹은 RPY 형식 모두 지원한다. 혹시, 이 설정이 직관적이지 않아 어렵다면 이 부분을 변경하여 활용할 수 있다.

<그림 Ⅸ-4> TCP 정량 위치 이동 설정

나 조인트 위치의 정량 이동

실무적으로 이동할 때, TCP의 위치보다는 특정 조인트의 회전만을 조정하고 싶은 경우가 있다. 프리드라이브를 활용하면 해당 조인트 이외에 다른 조인트가 함께 영향을 받아 개별적인 정밀 제어는 쉽지 않다. 이 경우 사용할 수 있는 기능이 <그림 IX-5>의 조인트 위치 정량 이동 기능이다.

이 기능의 첫 번째 활용 방법은 <그림 IX-5> (a) ①의 화살표를 누르는 방법이다. 이 방법은 누름과 동시에 로봇이 실제로 반응하므로 직관적인 장점이 있지만, 프리드라이브와 마찬가지로 정밀하게 정량 이동은 어렵다. 단위는 도 단위로 되어 편리하다.

TCP의 정량 이동과 마찬가지로 사용자가 직접 숫자로 입력하여 제어하는 방법도 제공하는데, <그림 IX-5> (a) ①의 숫자 부분을 클릭하면 <그림 IX-5> (b)와 같이 제어 화면이 나온다. 사용 방법은 TCP 정량 이동과 매우 유사하다. 입력 단위가 도 단위임을 주의하자. TCP 정량 이동의 설명과 마찬가지로 화면에 표시할 수 있는 크기의 한계로 직접 숫자를 입력하면 소수점 두 자리 정도가 한계이다. 따라서 특정 각도만큼을 회전해야 하는 경우 <그림 IX-5> (c)에서와 같이 버튼을 이용하는 것이 유리하다.

<그림 Ⅸ-5> 조인트 정량 회전

1. 지금까지 대부분의 학생들이 프리드라이브에 의존해서 정밀하지 않은 웨이포인트 설정을 하였을 것이다.

2. 이번 단원의 목적은 상대 위치 이동에 대한 이해도 있지만, 이동 탭을 이용하여 수치를 이용한 정량적인 위치 설정 방법에 대해서 경험하게 하는 것도 큰 목적이다.

3. 또한, 상대 위치를 이동하는 과정에서 로봇이 도달할 수 없는 특이점을 많이 경험할 수 있다. 상대 위치 이 동을 사용할 때의 주의점이 될 것이다.

4. 정밀하지 않은 설정은 오각형을 그리는 반복 동작 과정에서 특정 방향으로 이동하는 것은 지적해야 한다.

5. 또한, 고정 위치로 제일 먼저 시작하는 로봇 프로그램에서 초기 위치로 이동할 때 사용자의 조작이 필요 했던 부분이 상대 위치로 변경되었을 때 없어지는 것을 보여주고, 그 이유를 함께 생각해 보면 좋다.

6. 학생들이 상대 위치 이동을 활용한 다른 응용 프로그램(예: 스위치 입력 시 상대적 위치 이동 등)을 창의 적으로 만들 수 있도록 유도하자.
